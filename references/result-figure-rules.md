# Result Figure Rules

## Principle

Academic result slides should make substantive patterns visible. The audience should not need to mentally decode coefficient tables during a live presentation.

## Preferred displays

Use predicted probabilities, predicted values, marginal effects, adjusted means, coefficient plots, contrast plots, event-study figures, or subgroup slope plots depending on the model and research question.

For binary or categorical outcomes, predicted probabilities are usually easier to understand than log-odds coefficients. For interactions, show the conditional pattern directly. For continuous outcomes, show predicted values or marginal effects with uncertainty intervals.

## One-result-per-slide rule

Each result slide should answer one question. If a slide contains more than one claim, split it. If a figure has more than four focal series, either simplify, facet, or move some material to backup.

## Headline rule

The slide headline must state the result in substantive language. Avoid titles such as “Model 2” or “Regression Results.” Use claims such as “Women’s higher earnings reduce expected domestic responsibility more than men’s higher earnings increase it.”

## Model note rule

Each result slide should include a compact note with enough information to evaluate the claim: sample, model family, key controls, uncertainty interval, and scale interpretation. The note should not become a methods paragraph.

## Interaction rule

Never present an interaction only as a coefficient. Show predicted values, marginal effects, pairwise contrasts, or another interpretable display. If the interaction term is non-significant but the predicted pattern is theoretically important, explain this clearly and avoid overclaiming.

## Regression table rule

Use tables only when a table is analytically superior. A table in the main deck should contain few rows and only the columns needed for the argument. Full regression output belongs in backup.

## Visual encoding rule

Use teal for the focal contrast, gray for reference or background series, and red-orange only for anomalies, exceptions, or negative cases. Use direct labels instead of legends where possible. Do not use rainbow palettes for ordinal or theoretical contrasts.

## Uncertainty rule

Show uncertainty when the result involves estimation. Use confidence intervals, credible intervals, standard error bars, or clear model notes. If uncertainty is hidden to reduce clutter, explain why in the note or move the more technical version to backup.
